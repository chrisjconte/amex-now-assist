Platform Foundation

Generative AI Controller

Every AI call in Now Assist flows through the GenAI Controller. It connects to LLM providers, manages prompt governance, enforces data privacy, and moderates content — regardless of which model is underneath.

Prompt Management
Templates, variable injection, versioning — centralized prompt governance across all skills and agents.
Sensitive Data Handler
PII/PHI masking before prompts reach any LLM. Enforced at the controller layer, not the model layer.
Now Assist Guardian
Real-time content moderation — toxicity, bias, and misinformation filtering on every response.
LLM Provider Spokes
Pre-built Integration Hub connectors per provider. Swap models without changing skills.

The GenAI Controller is the reason Now Assist has model flexibility without developer complexity. Skills and agents are written once. Which model executes them is a configuration decision — not a code change.

This abstraction also means governance is universal. PII masking, content filtering, and audit logging apply whether the underlying model is ServiceNow’s own LLM or a third-party frontier model.

Platform Foundation

AI Control Tower

Centralized command center for governing all AI assets — including non-ServiceNow AI. Auto-discovers from AWS Bedrock and Azure AI Foundry; stores in CMDB via CSDM v5. Platform-wide governance that applies to all AI activity regardless of product or model choice.

Inventory & Discovery
  • Automated AI asset catalog (agents, models, datasets, workflows)
  • Auto-discovers from ServiceNow, AWS Bedrock, Azure AI Foundry
  • CMDB-backed (CSDM v5) with deduplication
Risk & Compliance
  • EU AI Act conformity assessment, NIST AI RMF, ISO/IEC 42001
  • GDPR, HIPAA, CPRA mapping
  • Bias detection and fairness audits
Policy Enforcement
  • Real-time guardrails (block, log, redirect)
  • Role-based access control
  • Model provider approval with audit trail
  • Lifecycle gates: Intake → Assessment → Deployment → Retirement
