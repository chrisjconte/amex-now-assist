Enterprise AI, flexible by design

Your domain expertise stays on-platform. The LLM is interchangeable — the intelligence is not. Every AI call is encrypted, isolated, and deleted after inference.

◆
Value
Built-in AI engine, not bolted on. The platform is the intelligence — the LLM is the compute layer.
▾
  • Built-in AI engine — not bolted on. Powers end-to-end enterprise workflows natively from a single platform
  • Platform-native AI inherits decades of ServiceNow domain context — workflows, data models, tools, and institutional knowledge are the AI’s foundation
  • Flexible LLM layer underneath — Now LLM Service or any frontier model (Azure OpenAI, Claude, Gemini). The engine adapts; the platform intelligence stays constant
  • Per-skill model routing — different LLMs for different use cases, swappable without code changes or retraining
⚙
Architecture
All data stays within ServiceNow’s platform boundary. Private endpoints, regional processing, no customer tenant routing.
🔍 Click to view full architecture diagram
↗
🔒
Security
Zero persistence processing. Prompts exist only for the duration of the inference call, then they’re gone.
▾
  • Zero persistence processing — prompts and responses processed in memory only, deleted immediately after inference. No training, caching, or storage on any compute hub — Now LLM Service, Azure OpenAI, Gemini, and Claude alike
  • Customer data never used to train models — Azure OpenAI abuse monitoring disabled; Gemini and Claude use enterprise configurations that prevent model learning
  • Real-time anonymization — Data Privacy for Now Assist masks sensitive fields (names, emails, phone numbers) with tokens before any LLM call; original values restored only on return to the instance
  • No data commingling — customer data isolated per customer during inference, even on shared OEM tenancy. Domain-separated instances maintain strict logical isolation with no cross-domain persistence
